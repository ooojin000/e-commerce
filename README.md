# 백엔드 개발자 과제 - API 설계 및 구현

## 과제 개요
간단한 상품 주문 관리 시스템의 API를 설계하고 구현해주세요.
가상의 이커머스 서비스에서 사용자가 상품을 주문하고, 주문 내역을 조회하며, 주문을 취소할 수 있는 기능이 필요합니다.

> **안내**: 과제 진행 시 이 저장소를 Fork하지 말고, 상단의 **Use this template** 버튼을 사용해주세요.
> Star/Watch도 지원자 간 노출이 될 수 있으니 삼가해주세요.
---

## 기능 요구사항

### [필수]
1. **상품 목록 조회**
    - 등록된 상품 목록을 조회할 수 있어야 합니다.
    - **상품 정보**: 상품명, 가격, 재고 수량
2. **주문 생성**
    - 사용자가 상품을 주문할 수 있어야 합니다.
    - 주문 시 재고가 차감되어야 합니다.
    - 재고가 부족한 경우 적절한 예외 처리가 필요합니다.
3. **주문 내역 조회**
    - 사용자별 주문 내역을 조회할 수 있어야 합니다.
4. **주문 취소**
    - 주문을 취소하면 재고가 복구되어야 합니다.

### [선택사항] (가산점)
- **동시성 이슈 제어**: 동시에 여러 주문 요청이 들어올 때 재고 정합성을 보장하는 방법 고려
- 페이징 처리 (상품 목록, 주문 내역)
- API 문서화 (Swagger/RestDocs 등)
- 단위 테스트 또는 통합 테스트 코드 작성

## 구현 가이드 & 제약사항
과제 진행 시 아래 내용을 중점적으로 고려하여 구현해주세요.

- **인증/인가 생략**: 별도의 로그인 구현은 필요 없습니다. API 요청 시 **User ID(식별자)**를 Request Body 또는 Header로 전달받는다고 가정하고 구현해주세요.
- **초기 데이터**: 애플리케이션 실행 시 테스트 가능한 **기초 상품 데이터가 자동으로 입력되도록(`data.sql` 등)** 제공해주세요.
- **API 설계**: RESTful 원칙을 준수하고, 일관된 응답 구조(Response Wrapper 등)를 가져야 합니다.
- **예외 처리**: 비즈니스 로직(예: 재고 부족, 존재하지 않는 상품 등)에서 발생할 수 있는 에러 상황을 고려하여 적절히 처리해야 합니다.

## 기술 스택
- **필수**: Java 17+, Spring Boot 3.x
- **DB**: H2 (in-memory) 혹은 본인이 주로 사용하는 DB 
- **데이터 접근 기술**: JPA (Spring Data JPA) 또는 MyBatis 중 택 1
- **빌드**: Gradle 또는 Maven
- **그 외**: 라이브러리는 자유롭게 선택
